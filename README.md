## Why Celtrix?

Starting a new project should take seconds, not hours.

Every developer knows the pain: before writing a single line of business logic, you spend hours configuring tools, resolving dependency conflicts, setting up folder structures, writing boilerplate, and connecting everything together. By the time the setup is done, the motivation is gone.

Celtrix eliminates all of that. One command. Your stack. Your app - running instantly.

---

## Quick Start

No global installation required.

```bash
npx celtrix my-app
cd my-app
npm run dev
```

Your application will be running at `http://localhost:3000`.

Follow the interactive prompts to choose your stack, language, and configuration. The entire process takes under 60 seconds.

---

## How It Works

1. Run the CLI and choose your stack from the interactive menu.
2. Celtrix sets up the full project structure, installs dependencies, and configures your tooling.
3. Start coding immediately - no manual configuration required.

---

## What You Get

Every project generated by Celtrix includes a production-ready foundation:

- Clean, consistent folder structure with clear separation of concerns
- Backend and frontend pre-wired and ready to run
- ESLint configuration included out of the box
- API setup following best practices
- Sample UI components to build from
- Environment variable template (`.env.example`) pre-populated
- Auto-generated `README.md` specific to your project with run instructions, env variable descriptions, and deployment steps

## Supported Stacks

| Stack | Description |
|---|---|
| MERN | MongoDB, Express, React, Node.js |
| MEAN | MongoDB, Express, Angular, Node.js |
| MEVN | MongoDB, Express, Vue.js, Node.js |
| Next.js | Full-stack React with API routes |
| Hono | Lightweight, high-performance backend |

Each stack includes variants such as Tailwind CSS, authentication scaffolding, and Firebase integration.

## Contributing

Celtrix is built by the community, for the community. Contributions of all kinds are welcome - whether that is a bug report, a new template, a documentation improvement, or a first pull request.

To get started:

1. Fork the repository and clone your fork locally.
2. Check the [CONTRIBUTING.md](./CONTRIBUTING.md) for the full contribution guide, including how templates work, how to add a new stack, and how to run the CLI locally.
3. Browse issues tagged `good first issue` or `help wanted` for a starting point.
4. Open a pull request with a clear description of what you changed and why.

All pull requests are reviewed and acknowledged within 48 hours.
